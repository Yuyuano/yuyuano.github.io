# 【洛谷 B2669】金币（NOIP 2015 普及组 T1）

[[原题跳转]](https://www.luogu.com.cn/problem/B2669)

---

## 题目一句话
国王发工资：
- 第 1 天给 1 枚；
- 接下来连续 2 天，每天给 2 枚；
- 再接下来连续 3 天，每天给 3 枚；
- ……

问前 `k` 天一共拿到多少金币？  
数据范围：`1 ≤ k ≤ 10 000`，随便模拟。

---

## 举个栗子（k = 6）

| 天数区间 | 每天金币 | 发了几天 | 小计 |
| --- | --- | --- | --- |
| 1 | 1 枚 | 1 天 | 1×1 = 1 |
| 2~3 | 2 枚 | 2 天 | 2×2 = 4 |
| 4~6 | 3 枚 | 3 天 | 3×3 = 9 |
| **总计** |  |  | **14 枚** |

---

## 思路：把“大块”当成一个整体
把相同金币的连续天数看成**一整块**，一块一块地啃：

1. 当前块应该发 `i` 枚金币，最多连发 `i` 天；
2. 剩余天数 `k` 不足 `i` 天，就**有多少天算多少天**；
3. 金币 += 天数 × 金币数，剩余天数减掉，进入下一块 `i+1`。

### Mermaid 流程图（可缩放）
```mermaid
flowchart TD
    A([开始]) --&gt; B[输入总天数 k]
    B --&gt; C[total = 0&lt;br&gt;i = 1]
    C --&gt; D{k &gt; 0 ?}
    D -- 是 --&gt; E[days = min(k, i)]
    E --&gt; F[total += days × i]
    F --&gt; G[k -= days]
    G --&gt; H[i += 1]
    H --&gt; D
    D -- 否 --&gt; I[输出 total]
    I --&gt; J([结束])
```

## 代码实现
```python
k = int(input())      # 还要发多少天
total = 0             # 金币总和
i = 1                 # 当前块每天发 i 枚
while k > 0:          # 只要还有天数就继续
    days = min(k, i)  # 本轮实际发几天
    total += days * i # 累加金币
    k -= days         # 扣掉已发天数
    i += 1            # 进入下一块
print(total)
```
